# 脚本使用说明

本项目提供两个快捷脚本，分别用于**启动服务**和**提交代码**。每个脚本有两个版本：`.bat` 供 Windows 双击使用，`.sh` 供 Git Bash / macOS / Linux 终端使用。

---

## start.bat / start.sh — 一键启动

**作用**：同时启动前端（Vite）和后端（Express）服务。

### 使用方法

- **Windows**：双击 `start.bat`
- **终端**：`./start.sh`

### 启动后访问

| 服务 | 地址 |
|------|------|
| 前端页面 | http://localhost:5173 |
| 后端 API | http://localhost:3001/api |

### 说明

- **首次运行**：脚本会自动安装前后端依赖（`npm install`）并初始化数据库，无需手动操作。
- **Windows**：会弹出两个独立命令窗口，分别显示前端和后端的实时日志，关闭窗口即停止对应服务。
- **终端（.sh）**：前后端在同一终端后台运行，按 `Ctrl+C` 同时停止所有服务。

---

## push.bat / push.sh — 一键提交 GitHub

**作用**：将本地所有改动提交并推送到 GitHub 仓库（`LEOliyutian/AVA`，`main` 分支）。

### 使用方法

- **Windows**：双击 `push.bat`
- **终端**：`./push.sh`

### 操作流程

1. 显示当前所有改动文件
2. 提示输入提交说明
   - 输入内容后回车 → 使用你输入的内容
   - 直接回车 → 自动生成（格式：`update: YYYY-MM-DD HH:MM 提交`）
3. 自动执行 `git add -A` → `git commit` → `git push origin main`

### 注意

- 推送前请确保已登录 GitHub（首次使用需在浏览器或终端完成 GitHub 身份认证）。
- 若没有任何改动，脚本会提示"无需提交"并退出。

---

## 手动启动（备用）

如果脚本无法运行，可以手动在两个终端中分别执行：

```bash
# 终端 1 — 后端
cd server
npm run dev

# 终端 2 — 前端
npm run dev
```

首次运行需先初始化数据库：

```bash
cd server
npm run db:init
```
